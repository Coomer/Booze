booze

A debug bar for rails

I still write traditional Rails apps (not just using Rails as an API for the frontend), and when doing so find it useful to immediately see a query count on generated pages. This allows me to easily detect n+1 queries.

This is where booze, a ruby gem I wrote for this, comes into play.

How it Works

  1. A railtie subscribes to ActiveSupport::Notifications to save queries in memory as a controller action runs. Queries are cleared when the action is done performing.
  2. The railtie adds an after_filter to display a debug bar right before the closing </body> tag.
  3. This debug bar has the total query count and can be expanded to show all queries used to render the page.
